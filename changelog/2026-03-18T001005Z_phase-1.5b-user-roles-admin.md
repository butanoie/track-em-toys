## Summary

### Guards & Security

- No self-modification (UUID case-insensitive comparison)
- No escalation above own role level
- Last-admin protection (`FOR UPDATE` serialized count check)
- GDPR-purged user rejection on all mutation routes (409)
- Content-Type enforcement hook on admin plugin
- ILIKE wildcard escaping (`\`, `%`, `_`) in email search
- UUID format validation on all `:id` params

### JWT Changes

- `signAccessToken` now embeds `{ sub, role }` in JWT payload
- `FastifyJWT` type augmentation extended to include `role: UserRole`
- Token refresh fetches current role via `getUserStatusAndRole()` (single merged query)
- Pre-migration tokens (no role claim) â†’ 403 via `isRolePayload` guard

### GDPR Purge Sequence (single transaction)

1. Scrub PII: email, display_name, avatar_url â†’ NULL
2. Set tombstone flags: `deleted_at = NOW()`, `deactivated_at = COALESCE(deactivated_at, NOW())`
3. Hard-delete: `oauth_accounts`, `refresh_tokens`
4. Scrub audit PII: `auth_events` ip_address, user_agent, metadata â†’ NULL
5. Log `user_purged` audit event with actor metadata

### CLI Bootstrap

## Technical Details

### Role Hierarchy

```typescript
export const ROLE_HIERARCHY: Record<UserRole, number> = { user: 0, curator: 1, admin: 2 };
// requireRole('curator') grants access to curators AND admins
```

### Fastify Decorator Pattern

```typescript
// server.ts
fastify.decorate('requireRole', requireRole);

// Route usage
preHandler: [fastify.authenticate, fastify.requireRole('admin')];
```

### Last-Admin Race Prevention

`findUserForAdmin` uses `SELECT ... FOR UPDATE` to serialize concurrent mutations. Two simultaneous admin demotions both lock the target row â€” the second blocks until the first commits, then sees the post-commit state.
