# Phase 1.7 Slice 4 — Character & Item Detail Pages

**Date:** 2026-03-19
**Time:** 22:04:08 UTC
**Type:** Feature
**Phase:** 1.7 Web Catalog Browsing UI
**Issue:** #61

## Summary

Added standalone character and item detail pages to the web catalog. Characters display sub-groups, appearances, combiner relationships, and related items. Items display a photo gallery with lightbox, linked metadata (character, manufacturer, toy line), and a share link. Both pages share content components with their respective sidebar panels, and the `CharacterStubPanel` placeholder was replaced with a fully functional `CharacterDetailPanel`.

---

## Changes Implemented

### 1. Character Detail Page

New standalone page at `/catalog/:franchise/characters/:slug` displaying full character profile with faction, continuity family, type, alt-mode, combiner info (cross-linked to gestalt/component characters), sub-groups (badges, hidden when empty), appearances table (Name/Source/Years), and related items list fetched via a new `character` filter on the items API.

### 2. Item Detail Page

New standalone page at `/catalog/:franchise/items/:slug` sharing `ItemDetailContent` with the sidebar panel. Features a photo gallery with Dialog-based lightbox (arrow key navigation, focus trap, WCAG-compliant), linked metadata (character → character page, manufacturer → hub, toy line → filtered items), and a clipboard share link button.

### 3. Shared Component Architecture

Extracted reusable components to eliminate duplication between panels and pages:

- `DetailPanelShell` — panel chrome (aside, focus management, Escape key, loading/error/empty states, close button)
- `DetailField` — extended with `children` prop for linked/badged content
- `ItemDetailContent` / `CharacterDetailContent` — pure display components
- `PhotoGallery` / `AppearancesTable` / `ShareLinkButton` — focused UI components

### 4. API: Character Filter on Items Endpoint

Added `character?: string` filter to `buildItemsQuery` in franchise-scoped items routes. One WHERE clause addition (`ch.slug = $N`) — no new JOIN needed since characters is already INNER JOINed. Added to both list and facets querystring schemas.

### 5. Search Panel Upgrade

Replaced `CharacterStubPanel` ("coming soon" placeholder) with `CharacterDetailPanel` that fetches real character data via `useCharacterDetail` hook. Panel now shows full character info with a "View full profile" link.

### 6. Cross-Cutting Improvements

- All 9 catalog pages: `<h2>` → `<h1>` for primary page heading (WCAG 2.4.6)
- `BaseItemFilters` shared type between `ItemFilters` and `ManufacturerItemFilters`
- Route restructure: `$franchise/items.tsx` → `$franchise/items/index.tsx` to support `items/$slug.tsx` without layout conflict
- Panel Escape handlers check `e.defaultPrevented` to prevent double-fire when Dialog lightbox is open
- Fixed `idx` increment bug in `buildItemsQuery` for `is_third_party` filter

---

## Technical Details

### Route Structure

TanStack Router file-based routing required moving `items.tsx` to `items/index.tsx` to avoid treating it as a layout route when adding `items/$slug.tsx` as a sibling. The `characters/$slug.tsx` route had no conflict since no `characters.tsx` existed.

### Photo Lightbox

Uses Shadcn `Dialog` (Radix `@radix-ui/react-dialog`) for WCAG-compliant modal behavior: focus trap, body scroll lock, `aria-modal`, Escape to close. Arrow key navigation via `onKeyDown` on `DialogContent`. Prev/Next buttons disabled at boundaries (no wrap-around).

### Zod Schema

`CharacterDetailSchema` mirrors the API `characterDetail` Fastify schema exactly (16 fields). `CharacterAppearanceSchema` covers nested appearances array with nullable `source_media`, `source_name`, `year_start`, `year_end`, `description`.
