# Phase 1.5b-UI — Admin Web Dashboard

**Date:** 2026-03-18
**Time:** 17:54:32 UTC
**Type:** Feature Addition
**Phase:** 1.5b-UI
**Version:** v0.5.0

## Summary

Built the admin section of the web app under `/admin/*` routes with user management capabilities. The admin dashboard features a role-guarded layout with sidebar navigation, a data table for user management (search, filter, paginate), inline role assignment, deactivate/reactivate actions, and GDPR purge with type-to-confirm safeguard. All admin JS is automatically code-split and never ships to regular users.

## Technical Details

### Code Splitting

TanStack Router's `autoCodeSplitting: true` automatically lazy-loads route components:

- `admin-*.js` (4.5 kB gzip) — Layout, sidebar, role guard
- `users-*.js` (34 kB gzip) — Users page, table, dialogs, Radix primitives
- `index-*.js` (115 kB gzip) — Main bundle (unchanged for regular users)

### Architecture Decisions

1. **Component-level role guard** (not `beforeLoad`) — avoids race condition where `authStore.getToken()` is null during cold-load silent refresh
2. **URL search params** for filter/pagination state — bookmarkable, survives refresh, syncs with browser history
3. **`throwApiError` shared helper** — eliminates error-handling duplication between `apiFetchJson` and void-response endpoints
4. **All destructive actions through `ConfirmDialog`** — role changes, deactivation, reactivation, and GDPR purge all use the generic dialog pattern

### Shadcn CLI Path Fix

The CLI v4.0.8 doesn't resolve TypeScript path aliases (`@/` → `src/`) when determining output paths. Fixed `components.json` aliases from `@/...` to `src/...`.
