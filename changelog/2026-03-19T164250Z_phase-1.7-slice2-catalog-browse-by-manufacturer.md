# Phase 1.7 Slice 2 — Catalog Browse by Manufacturer

**Date:** 2026-03-19
**Time:** 16:42:50 UTC
**Type:** Feature Addition
**Phase:** 1.7
**Version:** v0.7.1

## Summary

Implemented the second vertical slice of the web catalog browsing UI — browsing by manufacturer. Three new pages provide manufacturer-centric catalog navigation: manufacturer list (grid/table toggle), manufacturer hub (metadata, franchise/toy line cards, stats), and manufacturer items (three-column layout with faceted filtering, keyboard navigation, and detail panel). Three new API endpoints support the UI: manufacturer stats, manufacturer-scoped items with cursor pagination, and cross-filtered facet counts with franchise dimension replacing manufacturer.

## Changes Implemented

### 1. API — Manufacturer Stats Endpoint

New `GET /catalog/manufacturers/stats` endpoint returns item, toy line, and franchise counts per manufacturer using a subquery JOIN pattern to avoid Cartesian product between items and toy_lines tables.

**Modified:**

- `api/src/catalog/manufacturers/queries.ts` — added `ManufacturerStatsRow`, `listManufacturerStats()` with subquery pattern
- `api/src/catalog/manufacturers/schemas.ts` — added `manufacturerStatsItem`, `listManufacturerStatsSchema`
- `api/src/catalog/manufacturers/routes.ts` — registered `GET /stats` before `/:slug`

### 2. API — Manufacturer-Scoped Items List

New `GET /catalog/manufacturers/:slug/items` endpoint provides cursor-paginated items scoped to a manufacturer with 5 filter params: franchise, size_class, toy_line, continuity_family, is_third_party. Uses `buildManufacturerItemsQuery()` — a parallel query builder to the franchise-scoped `buildItemsQuery()`.

Key differences from franchise-scoped items:

- WHERE starts with `mfr.slug = $1` (not `fr.slug = $1`)
- INNER JOIN on manufacturers (not LEFT JOIN, since items are scoped to a known manufacturer)
- INNER JOIN on franchises (always, since items.franchise_id is NOT NULL and SELECT needs fr.slug/fr.name)
- Filter dimensions: franchise replaces manufacturer

**Modified:**

- `api/src/catalog/manufacturers/queries.ts` — added `ManufacturerItemFilters`, `buildManufacturerItemsQuery()`, `listManufacturerItems()`
- `api/src/catalog/manufacturers/schemas.ts` — added `manufacturerItemsListQuerystring`, `listManufacturerItemsSchema`
- `api/src/catalog/manufacturers/routes.ts` — added `GET /:slug/items` with 404 validation

### 3. API — Manufacturer-Scoped Facets

New `GET /catalog/manufacturers/:slug/items/facets` endpoint returns cross-filtered facet counts. The response shape has `franchises[]` instead of `manufacturers[]` — a different Fastify response schema from franchise facets. Each facet dimension runs a parallel GROUP BY query excluding its own filter via `filtersExcluding(key)`.

**Modified:**

## Technical Details

### Subquery Stats Pattern

```sql
SELECT m.slug, m.name, ...,
       COALESCE(ic.item_count, 0)::int AS item_count,
       COALESCE(ic.franchise_count, 0)::int AS franchise_count,
       COALESCE(tlc.toy_line_count, 0)::int AS toy_line_count
  FROM manufacturers m
  LEFT JOIN (SELECT manufacturer_id, COUNT(*)::int AS item_count, ... FROM items GROUP BY manufacturer_id) ic ON ...
  LEFT JOIN (SELECT manufacturer_id, COUNT(*)::int AS toy_line_count FROM toy_lines GROUP BY manufacturer_id) tlc ON ...
```

Avoids Cartesian product between items and toy_lines that a dual LEFT JOIN would produce.

### No Manufacturer-Scoped Item Detail

Item slugs are `UNIQUE(slug, franchise_id)`, not per-manufacturer. A manufacturer-scoped detail endpoint (`WHERE mfr.slug = $1 AND i.slug = $2`) could match multiple items across franchises. Instead, `ItemDetailPanel` receives the franchise slug from `selectedItem.franchise.slug` in the list response.

### Generic FacetSidebar

```typescript
interface FacetGroupConfig {
  label: string;
  values: FacetValue[];
  filterKey: string;
  activeValue: string | boolean | undefined;
}
```

Each page constructs its own groups array, removing the sidebar's dependency on any specific filter type.

## Impact Assessment

- **Users:** Can now browse the toy catalog by manufacturer, complementing franchise-based browsing
- **Collectors:** Manufacturer-centric faceted search enables finding items by franchise, toy line, size class across a manufacturer's entire catalog
- **Architecture:** FacetSidebar refactor to generic `groups[]` enables future browsing scopes without component changes. Shared `itemListItem`/`facetValueItem` schemas and `formatListItem`/`formatDetail` formatters reduce duplication.
- **Bundle:** Auto code-splitting keeps manufacturer chunks separate (~6KB items page, ~5.6KB list page)
