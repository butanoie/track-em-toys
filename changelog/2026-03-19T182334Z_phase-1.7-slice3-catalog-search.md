# Phase 1.7 Slice 3 — Catalog Search UI

**Date:** 2026-03-19
**Time:** 18:23:34 UTC
**Type:** Feature
**Phase:** 1.7 (Web Catalog Browsing UI)
**Version:** v0.7.3

## Summary

Implemented catalog search (Slice 3 of Phase 1.7), adding full-text search across characters and items with enriched results, a site-wide search input in the app header, grouped search results page, and detail panels for both item and character results.

## Changes Implemented

### API Enrichment

- Extended the `GET /catalog/search` endpoint to return all `CatalogItem` fields for item results
- Added 3 JOINs to the items UNION branch: `JOIN characters`, `LEFT JOIN manufacturers`, `JOIN toy_lines`
- Character UNION branch pads with typed NULL literals to keep column alignment
- Count query intentionally unchanged (counts only need `search_vector` match)
- Removed redundant `Math.max/min` limit clamping (Fastify schema already validates)
- Fastify response schema extended with 7 new nullable fields, all in `required` array

### Web Data Layer

- `SearchCharacterResultSchema` and `SearchItemResultSchema` using Zod discriminated union on `entity_type`
- `SearchItemResultSchema = CatalogItemSchema.extend({ entity_type: z.literal('item') })` — zero-maintenance schema inheritance
- `searchCatalog()` API client function with URLSearchParams construction
- `useSearch` TanStack Query hook with `enabled: q.trim().length >= 1`, `keepPreviousData`, `staleTime: 30_000`

### Web UI Components

- **SearchInput** — compact input in AppHeader with 300ms debounce, `replace: true` during debounce, `replace: false` on Enter. Syncs with URL on back navigation. Clear button and Escape key both navigate to clear the URL.
- **SearchPage** — groups results by entity type (Characters section, Items section). Reuses `ItemList` for items. `selectResult` helper eliminates duplicate navigation logic.
- **CharacterResultList** — keyboard-navigable character results with arrow keys and Escape
- **CharacterStubPanel** — "Character detail pages coming soon" stub with consistent ARIA structure
- **SearchPagination** — numbered page controls with Previous/Next, max 7 visible pages with ellipsis

### Route & Integration

- TanStack Router route at `/_authenticated/catalog/search` with search params: `q`, `page`, `selected`, `selected_type`
- AppHeader updated to render `SearchInput` on all authenticated pages (admin pages unaffected — they use their own header)
- Existing AppHeader and SettingsPage test mocks updated for `useNavigate` and `useRouterState`

## Technical Details

### Zod Discriminated Union

```typescript
export const SearchResultSchema = z.discriminatedUnion('entity_type', [
  SearchCharacterResultSchema, // lean: name, slug, franchise
  SearchItemResultSchema, // full CatalogItem + entity_type
]);
```

This ensures type-safe narrowing at the component boundary — `items` array is structurally compatible with `CatalogItem[]` without explicit casts.

### Debounce Pattern

SearchInput uses `useRef<setTimeout>` with `clearTimeout` on every keystroke. Debounced navigations use `replace: true` (no history pollution). Enter submits immediately with `replace: false` (proper history entry).

### Detail Panel Franchise Resolution

Item detail panel receives `franchise` from `selectedItem?.franchise.slug` (looked up from the results array), matching the `ManufacturerItemsPage` pattern. `useItemDetail` is disabled when franchise is empty string.
