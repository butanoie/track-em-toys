# Phase 1.7 Slice 1 — Catalog Browsing UI (By Franchise)

**Date:** 2026-03-19
**Time:** 06:26:42 UTC
**Type:** Feature Addition
**Phase:** 1.7
**Version:** v0.7.0

## Summary

Implemented the first vertical slice of the web catalog browsing UI — browsing by franchise. Three new pages replace the placeholder dashboard with a real catalog browser: franchise list (grid/table toggle), franchise hub (overview with CF and manufacturer cards), and items list (three-column layout with faceted filtering, keyboard navigation, and detail panel). Three new API endpoints support the UI: franchise stats, item filter params, and cross-filtered facet counts.

## Changes Implemented

### 1. API — Franchise Stats Endpoint

New `GET /catalog/franchises/stats` endpoint returns item, continuity family, and manufacturer counts per franchise in a single GROUP BY query. JOINs through `items → characters → continuity_families` to count only entities with actual items.

**Modified:**

- `api/src/catalog/franchises/queries.ts` — added `FranchiseStatsRow`, `listFranchiseStats()`
- `api/src/catalog/franchises/schemas.ts` — added `franchiseStatsItem`, `listFranchiseStatsSchema`
- `api/src/catalog/franchises/routes.ts` — registered `GET /stats` endpoint
- `api/src/catalog/franchises/routes.test.ts` — added stats endpoint integration tests

### 2. API — Items List Filter Params

Extended `GET /catalog/franchises/:franchise/items` with 5 optional filter query params: `manufacturer`, `size_class`, `toy_line`, `continuity_family`, `is_third_party`. Uses a dynamic param builder (`buildItemsQuery`) that produces a shared WHERE clause for both data and count queries, ensuring identical filter conditions per `api/CLAUDE.md` rules.

**Modified:**

- `api/src/catalog/items/queries.ts` — added `ItemFilters` interface, `buildItemsQuery()` helper, extended `ListItemsParams`
- `api/src/catalog/items/schemas.ts` — created `itemsListQuerystring` (separate from shared `paginationQuery` due to `additionalProperties: false`)
- `api/src/catalog/items/routes.ts` — thread filter params via `extractFilters()`, register facets route before `/:slug`
- `api/src/catalog/items/routes.test.ts` — added filter and facets integration tests

### 3. API — Facet Counts Endpoint

New `GET /catalog/franchises/:franchise/items/facets` endpoint returns cross-filtered facet counts. Each facet dimension (manufacturer, size_class, toy_line, continuity_family, is_third_party) runs a parallel GROUP BY query that excludes its own filter, giving standard faceted search UX. All facets use unified `{ value, label, count }` shape.


### Cross-Filtered Facets

Each facet dimension runs its own GROUP BY query via `Promise.all`:

- Manufacturer facet: applies all filters EXCEPT manufacturer
- Size class facet: applies all filters EXCEPT size_class
- etc.

This gives users the standard "show all options with counts reflecting other filters" UX.

### URL-Driven State

All filter state lives in URL search params via TanStack Router's `validateSearch`. Filter changes clear cursor and selected item. Browser back/forward works correctly.

### Boolean Search Param Fix

`z.coerce.boolean()` was replaced with `z.enum(['true','false']).transform(v => v === 'true')` because `Boolean("false")` returns `true` in JavaScript.
