# Seed Data

Database seed files for the shared catalog tables defined in migrations `011_shared_catalog_tables.sql`, `012_enrich_characters_table.sql`, and `013_continuity_families_and_appearances.sql`.

## Data Sources

Seed data comes from one of two locations:

- **Sample data** (`api/db/seed/sample/`) — Minimal FK-consistent fixtures included in the monorepo. Used by default for development and CI.
- **Full catalog data** (external repo) — Proprietary dataset in a separately licensed private repository. Activated by setting `SEED_DATA_PATH` to the external repo's root directory.

When `SEED_DATA_PATH` is unset, the ingest script and validation tests default to `sample/`.

### External Data Repo Layout

The external repo must mirror the `sample/` directory structure:

```
track-em-toys-data/
├── reference/           (franchises, factions, sub_groups, etc.)
├── characters/          (character JSON files per continuity)
├── appearances/         (character appearance JSON files)
├── relationships/       (entity relationship JSON files)
└── items/               (item files in manufacturer subdirs)
```

## Import Order

Tables must be seeded in dependency order:

| Order | Table               | File(s)                                | References                                                                      |
|-------|---------------------|----------------------------------------|---------------------------------------------------------------------------------|
| -1    | franchises          | `reference/franchises.json`            | —                                                                               |
| 0     | continuity_families | `reference/continuity_families.json`   | franchises (franchise_slug)                                                     |
| 1     | factions            | `reference/factions.json`              | franchises (franchise_slug)                                                     |
| 2     | sub_groups          | `reference/sub_groups.json`            | factions (faction_slug), franchises (franchise_slug)                            |
| 3     | manufacturers       | `reference/manufacturers.json`         | —                                                                               |
| 4     | toy_lines           | `reference/toy_lines.json`             | manufacturers (manufacturer_slug), franchises (franchise_slug)                  |
| 5     | characters          | `characters/*.json`                    | factions (faction_slug), continuity_families (continuity_family_slug), franchises (franchise_slug) |
| 5b    | character_sub_groups | `characters/*.json` (sub_group_slugs) | characters, sub_groups (via sub_group_slugs array)                              |
| 5.5   | character_appearances | `appearances/*.json`                 | characters (character_slug)                                                     |
| 5.7   | character_relationships | `relationships/*.json`             | characters (entity1.slug, entity2.slug)                                         |
| 6     | items               | `items/*.json`                         | manufacturers, toy_lines, characters, character_appearances (all via slugs)     |

## FK Convention: Slugs, Not IDs

All foreign key references use **slug strings**, never integer or UUID IDs.

```jsonc
// CORRECT — stable, self-documenting
{ "faction_slug": "autobot", "sub_group_slug": "dinobots" }

// WRONG — positional, breaks on reorder/regeneration
{ "faction_id": 1, "sub_group_id": 1 }
```

The seed ingestion script resolves slugs to UUIDs at insert time (e.g., `INSERT INTO ... SELECT id FROM factions WHERE slug = $1`).

UUID primary keys are auto-generated by PostgreSQL (`DEFAULT gen_random_uuid()`) — never hardcode them in seed files.

## Slug Format

- Lowercase kebab-case: `optimus-prime`, `fanstoys-mainline`
- No apostrophes, periods, or special characters
- See migration 011 comments for the full convention

## Character Column Mapping

| Seed field | DB column | Notes |
|---|---|---|
| `name` | `name` | |
| `slug` | `slug` | |
| `franchise_slug` | `franchise_id` | Resolve via `SELECT id FROM franchises WHERE slug = $1` |
| `faction_slug` | `faction_id` | Resolve via `SELECT id FROM factions WHERE slug = $1` |
| `character_type` | `character_type` | Species or gimmick type (see migration 012 comment) |
| `sub_group_slugs` | `character_sub_groups` | Array of slugs → insert into junction table. Resolve each via `SELECT id FROM sub_groups WHERE slug = $1` |
| `alt_mode` | `alt_mode` | |
| `is_combined_form` | `is_combined_form` | Denormalized flag. Must be consistent with `combiner-component` relationships |
| `continuity_family_slug` | `continuity_family_id` | Resolve via `SELECT id FROM continuity_families WHERE slug = $1`. Added in migration 013. Always populated |
| `notes` | `metadata.notes` | Into JSONB |
| `series_year` | `metadata.series_year` | Into JSONB |

## Appearance Column Mapping

| Seed field | DB column | Notes |
|---|---|---|
| `slug` | `slug` | Globally unique, format: `{character-slug}-{source-descriptor}` |
| `name` | `name` | Display name, e.g., "Optimus Prime (G1 Cartoon)" |
| `character_slug` | `character_id` | Resolve via `SELECT id FROM characters WHERE slug = $1` |
| `description` | `description` | Brief visual description of the appearance |
| `source_media` | `source_media` | One of: TV, Comic/Manga, Movie, OVA, Toy-only, Video Game |
| `source_name` | `source_name` | Specific media title (e.g., "The Transformers Season 1") |
| `year_start` | `year_start` | Year the media began |
| `year_end` | `year_end` | Year the media ended (null if ongoing or single year) |
| `metadata` | `metadata` | JSONB for additional structured data |

## Purge and Re-seed

```bash
npm run seed:purge    # TRUNCATE all catalog tables + re-seed from SEED_DATA_PATH or sample/
npm run seed          # Upsert mode (no truncate)
```
