# Seed Data

Database seed files for the shared catalog tables defined in migrations `011_shared_catalog_tables.sql`, `012_enrich_characters_table.sql`, and `013_continuity_families_and_appearances.sql`.

## Import Order

Tables must be seeded in dependency order:

| Order | Table               | File(s)                                | References                                                                      |
|-------|---------------------|----------------------------------------|---------------------------------------------------------------------------------|
| 0     | continuity_families | `reference/continuity_families.json`   | —                                                                               |
| 1     | factions            | `reference/factions.json`              | —                                                                               |
| 2     | sub_groups          | `reference/sub_groups.json`            | factions (faction_slug)                                                         |
| 3     | manufacturers       | `reference/manufacturers.json`         | —                                                                               |
| 4     | toy_lines           | `reference/toy_lines.json`             | manufacturers (manufacturer_slug)                                               |
| 5     | characters          | `characters/*.json`                    | factions (faction_slug), characters (combined_form_slug), continuity_families (continuity_family_slug) |
| 5b    | character_sub_groups | `characters/*.json` (sub_group_slugs) | characters, sub_groups (via sub_group_slugs array)                              |
| 6     | items               | `items/*.json` (when added)            | manufacturers, toy_lines, characters, character_appearances (all via slugs)     |

## FK Convention: Slugs, Not IDs

All foreign key references use **slug strings**, never integer or UUID IDs.

```jsonc
// CORRECT — stable, self-documenting
{ "faction_slug": "autobot", "sub_group_slug": "dinobots" }

// WRONG — positional, breaks on reorder/regeneration
{ "faction_id": 1, "sub_group_id": 1 }
```

The seed ingestion script resolves slugs to UUIDs at insert time (e.g., `INSERT INTO ... SELECT id FROM factions WHERE slug = $1`).

UUID primary keys are auto-generated by PostgreSQL (`DEFAULT gen_random_uuid()`) — never hardcode them in seed files.

## Slug Format

- Lowercase kebab-case: `optimus-prime`, `fanstoys-mainline`
- No apostrophes, periods, or special characters
- See migration 011 comments for the full convention

## Character Files

Characters are split by source material to keep files manageable:

| File | Source | Characters |
|------|--------|-----------|
| `g1-season1.json` | NA cartoon Season 1 (1984) | 59 |
| `g1-season2.json` | NA cartoon Season 2 (1985-86) | 70 |
| `g1-movie.json` | The Movie (1986) | 18 |
| `g1-season3.json` | NA cartoon Season 3 (1986-87) | 54 |
| `g1-season4.json` | NA cartoon Season 4 "The Rebirth" (1987) | 43 |
| `g1-toy-only.json` | Hasbro G1 toys with no animated appearance | 92 |
| `jp-headmasters.json` | JP Headmasters anime (1987-88) | 21 |
| `jp-masterforce.json` | JP Masterforce anime (1988-89) | 30 |
| `jp-victory.json` | JP Victory anime (1989) | 38 |
| `jp-zone.json` | JP Zone OVA (1990) | 13 |

Characters are **not duplicated** across files — each file contains only NEW characters for that source.

## Purge and Re-seed

To purge and re-seed, drop and recreate the catalog tables (migration 011 has `migrate:down`), then re-import in the order listed above.
