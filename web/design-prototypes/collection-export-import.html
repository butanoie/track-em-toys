Component Architecture Notes

New Components

  • ExportImportToolbar — Button group, triggers export download and opens import dialog
  • ImportCollectionDialog — Multi-step dialog (upload → preview → results)
  • ImportDropZone — Drag-and-drop file target with validation
  • ImportPreview — File summary card with version, date, franchise breakdown
  • ImportResultsManifest — Scrollable ledger with success/failure items

Integration Points

  • Toolbar sits in CollectionPage.tsx line 149, between ViewToggle and pagination
  • Export calls GET /collection/export → triggers browser download via blob URL
  • Import calls POST /collection/import → returns structured results
  • On import success, invalidate ['collection'] query prefix
  • Reuses existing ConditionBadge for manifest item conditions
  • File parsing (JSON + schema version check) happens client-side before API call

Accessibility

  • Drop zone: role="button" with keyboard activation (Enter/Space)
  • File input: hidden <input type="file" accept=".json"> triggered by drop zone click
  • Import button: aria-disabled when no file selected, aria-busy during import
  • Results manifest: aria-live="polite" on summary counters
  • Error messages: role="alert" for validation failures

State Machine

  • idle → Drop zone shown, import button disabled
  • file-selected → Preview shown, import button enabled
  • importing → Loading spinner, buttons disabled
  • complete → Results manifest shown
  • error → Inline error in drop zone area
