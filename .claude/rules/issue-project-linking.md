After creating a GitHub issue with `gh issue create`, ALWAYS link it to the project board and populate its fields.

1. Add to project and capture the item ID (see `docs/references/gh-project-fields.md` for commands and all field/option IDs)
2. Ask the user to confirm values for: Status (default Todo), Priority, Phase, Track, Effort
3. Set each field using the IDs from the reference doc
4. Also set labels and milestone on the issue itself

Read `docs/references/gh-project-fields.md` before executing any project-linking commands.
