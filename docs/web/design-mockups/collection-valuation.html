Reading the design
Currency formatting (locale + ISO)
Symbol position follows the user's locale (English: $279.99; German: 279,99 €), and the ISO code always trails as a smaller muted suffix to disambiguate (CAD vs USD vs HKD all use $). Production code uses Intl.NumberFormat with currencyDisplay: 'narrowSymbol', then renders the ISO code as a separate trailing element.
Three tiers, three weights
Market Value is the largest number (insurance is the primary use case). MSRP and You Paid are equal secondary tiers. The serif Fraunces display font is used ONLY on the hero Market Value — treats the insurance-relevant number like a signature figure in a financial statement.
Coverage transparency
Every tier shows "X of Y items" and the summary card exposes an overall coverage percentage bar. This is load-bearing trust: collectors need to know the estimate is incomplete if only half their items have data.
FX disclaimer as first-class UI
The amber-bordered notice is not a footnote — it's positioned directly under the hero so users cannot miss it. For insurance documentation, the per-item observed amounts are the authoritative source; the aggregate is a convenience.
Per-item row honesty
"not entered" and "no data" are italicized muted text, not dashes — tells the user whether data is missing because they haven't entered it (actionable) or because the source didn't have it (not actionable). Different wording, different user response.
Delta chips (gain/loss)
Gain/loss shown only vs "You Paid" (the tier the user directly controls). Intentionally not shown vs MSRP — that comparison is less meaningful for collectors since many pay above MSRP for secondary-market items.
