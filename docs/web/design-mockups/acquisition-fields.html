Reading the design
Section divider with label
Visually separates acquisition fields from condition/grade/notes without forcing a collapse. The "Acquisition Details" label uses the same uppercase-tracking-wider treatment as field labels in the catalog detail page — internal consistency across surfaces.
Currency defaults from profile
Pre-fills with user.currency (cascaded from country if user hasn't set currency explicitly). The "from your profile" check-icon hint reinforces the connection. If user changes it for this item, the hint becomes "overrides your profile (CAD)" in the acquisition accent color.
Symbol prefix on the price input
The $ in the muted prefix box mirrors the symbol-prefix convention from the valuation display. The number is monospace + right-aligned for ledger-like alignment. Symbol updates dynamically when the currency changes (CAD/USD → $, GBP → £, EUR → €, JPY → ¥).
Source as free-text
No autocomplete, no enum — collectors buy from too many places to enumerate. Placeholder seeds expectations ("e.g., eBay, Hasbro Pulse, gift"). Future iteration could autocomplete from the user's own historical sources.
Future date is a warning, not a block
Pre-orders are real — collectors record planned purchases before delivery. Hard validation would frustrate this use case. The amber border + inline notice flags the unusual case but lets the save proceed.
"Clear" button on populated state only
Appears in the section header when any acquisition field has a value. Single-click resets all four fields back to defaults. Avoids forcing users to clear four fields individually if they want to remove a mistaken entry.
